# Card VT24H no Painel do BI-Gestão Técnica — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adicionar um card "VT24H" (Executou Prazo / Fora do Prazo / %) na aba Painel do BI-Gestão Técnica, usando o cálculo de cumprimento de prazo VT que já existe em `OSRow` (`_vtPrazoHoras`/`_vtCumpridaNoPrazo`, `src/lib/transform.ts`).

**Architecture:** Um builder puro novo (`buildVt24hStats`) filtra `allRows` (de `useOSDerived()`, mesmo hook usado no Mapa) por `_vtPrazoHoras === 24` e bucketiza por `dataexecucao` dentro do período local da página (`inicio`/`fim`, o mesmo estado que já alimenta `useBacklog`). `BiGestaoTecnicaPage` passa o resultado como prop nova pro `PainelTab`, que renderiza um card adicional no mesmo estilo dos já existentes.

**Tech Stack:** React + TypeScript, Vitest.

**Spec:** `.planning/specs/2026-07-23-vt24h-painel-design.md`

## Global Constraints

- Só VT24H — sem VT3H/VT4H (sem fonte de dado no sistema).
- `allRows` vem de `useOSDerived()` (contexto `OSDataContext`, já carregado globalmente pelo app) — não precisa de nova busca de rede.
- Bucketiza por `dataexecucao`, não `datacadastro` — o card mede cumprimento de prazo de OS **executadas**.
- `_vtCumpridaNoPrazo == null` (não executada) já exclui a OS naturalmente — não precisa checar `descsituacao` à parte.

- [ ] **Step 3: Criar `src/lib/builders/vt24h.ts`**

```ts
import { parseDate } from '../transform'
import type { OSRow } from '../types'

export interface Vt24hStats {
  executouPrazo:     number
  executouForaPrazo: number
  total:             number
  pctPrazo:          number
}

// inicio/fim vêm como strings ISO 'YYYY-MM-DD' (mesmo formato do estado de
// período de BiGestaoTecnicaPage). new Date('YYYY-MM-DD') parseia como UTC
// meia-noite — comparar direto contra parseDate() (que retorna Date local)
// causaria erro de fuso. Por isso parseamos inicio/fim como Date local aqui.
function parseIsoLocal(iso: string): Date {
  const [y, m, d] = iso.split('-').map(Number)
  return new Date(y, m - 1, d)
}

export function buildVt24hStats(allRows: OSRow[], inicio: string, fim: string): Vt24hStats {
  const iniDate = parseIsoLocal(inicio)
  const fimDate = parseIsoLocal(fim)

  let executouPrazo = 0
  let executouForaPrazo = 0

  for (const r of allRows) {
    if (r._vtPrazoHoras !== 24 || r._vtCumpridaNoPrazo == null) continue
    const dtExec = parseDate(r.dataexecucao)
    if (!dtExec || dtExec < iniDate || dtExec >= fimDate) continue
    if (r._vtCumpridaNoPrazo) executouPrazo++
    else executouForaPrazo++
  }

  const total = executouPrazo + executouForaPrazo
  return {
    executouPrazo,
    executouForaPrazo,
    total,
    pctPrazo: total > 0 ? Math.round((executouPrazo / total) * 100) : 0,
  }
}
```
